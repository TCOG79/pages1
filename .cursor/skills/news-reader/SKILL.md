# News Reader — PAGES 1 NEWS

## เมื่อไหร่ใช้

- User ให้ URL ข่าวหรือข้อความต้นทาง
- User ขอดึง/อ่านข่าวก่อนนำไปเผยแพร่
- ขั้นตอนแรกของ workflow: อ่าน → สรุป → เขียนไฟล์ → อ้างอิง

## ขั้นตอน

1. อ่านแหล่งต้นทาง (fetch URL หรืออ่านข้อความที่ user ให้)
2. ดึงข้อมูลดังนี้:

| รายการ | รายละเอียด |
|--------|------------|
| หัวข้อต้นทาง | ชื่อบทความเดิม |
| lead paragraph | ย่อหน้าแรกสำหรับ excerpt |
| ประเด็นหลัก | จุดสำคัญ 2–4 ข้อ สำหรับสรุป body |
| บุคคล/องค์กร | ใครเกี่ยวข้อง |
| วันที่เผยแพร่ | จากต้นทาง |
| หมวดข่าว | การเมือง / เศรษฐกิจ / กีฬา / บันเทิง / ไอที/เทคโนโลยี / ยานยนต์ / หวยไทย |
| รูปเด่น | og:image URL + ประเมิน `imageRights` |
| แหล่งอ้างอิง | ชื่อสำนักข่าว + URL บทความ |

3. **ไม่คัดลอกบทความเต็มลงไฟล์** — ส่งต่อให้ `news-summarizer` สร้างสรุป 2–3 ย่อหน้า
4. **ไม่เขียนไฟล์ `.md` เอง** — เว้นแต่ user ขอให้เขียนเลย

## Output Format

```markdown
## ข้อมูลข่าวจากต้นทาง

**หัวข้อต้นทาง:** ...
**แหล่ง:** [ชื่อ](URL)
**วันที่:** ...
**หมวดที่แนะนำ:** ...

### Lead (สำหรับ excerpt)

[ย่อหน้าแรก]

### ประเด็นหลัก (สำหรับสรุป body)

1. ...
2. ...
3. ...

### รูปภาพ
- og:image URL: ...
- imageRights: source_thumbnail / press_release / placeholder
- imageCredit: ภาพ: {sourceName}
- รัน: npm run fetch-image -- {slug} {sourceUrl} [imageRights]
```

## ข้อควรระวัง

- ระบุแหล่งต้นทางชัดเจนเสมอ
- ถ้า URL เปิดไม่ได้ แจ้ง user และขอข้อความต้นทางแทน
- ตรวจ ToS ของแหล่งข่าวก่อนดึงข้อมูล
